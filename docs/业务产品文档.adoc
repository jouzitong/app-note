== 2. 页面功能与边界（逐页定义）

=== P-001 首页（基于 `static-product/index.html`）

==== 页面定位

静态产品首页，当前阶段只实现“模块卡片列表”能力。

==== 必须实现功能

- 模块卡片区展示 4 个模块卡片。
- 模块卡片包含：图标、标题、描述、最近更新时间。
- 至少保留 1 个可跳转模块入口（当前是“日语学习笔记”）。
- 其余模块入口允许占位，后续再逐步实现。

==== 禁止实现功能（边界）

- 不实现搜索能力（搜索区仅占位或可隐藏）。
- 不做真实搜索请求与搜索结果联动。
- 不做待办编辑、删除、保存。
- 不做最近访问写入与排序。
- 不接入后端接口。

==== 输入字段（静态）

[cols="1,3",options="header"]
|===
| 字段 | 说明
| `modules[]` | 模块卡片列表（图标、标题、描述、更新时间、跳转路径）
| `searchPlaceholder` | 搜索框占位文案（当前不实现搜索）
| `todoPlaceholder` | 待办区域占位标识（当前不实现待办）
| `recentPlaceholder` | 最近访问区域占位标识（当前不实现最近访问）
| `footerText` | 页脚占位说明文案
|===

==== 输出动作

- 点击“日语学习笔记”模块卡片：跳转 `./language-jp.html`（当前页面内定义）。
- 点击其他 3 个模块卡片：当前为占位动作（`javascript:;`）。
- 其他区域仅展示，无提交类动作。

==== 验收条件

- 页面加载后模块卡片区可正常展示。
- 4 个模块卡片展示信息完整。
- 仅存在 1 个真实跳转入口，其余入口保持占位。
- 除模块卡片外的其他功能均不进入实现范围。

== 3. 页面交互逻辑（用户怎么走）

=== 主流程

1. 用户进入 `P-001` 首页。
2. 用户查看模块卡片区。
3. 用户点击“日语学习笔记”进入下一页面；或停留在首页。
