== 2. 页面功能与边界（逐页定义）

=== P-001 首页

原型来源：`static-product/index.html`

==== 页面定位

App Note 静态首页，承担“模块入口聚合 + 待办展示 + 最近访问展示”。

==== 必须实现功能

- 顶部展示搜索输入框（仅占位展示）。
- 模块卡片区展示 4 个模块卡片，每个卡片包含：图标、标题、描述、最近更新时间。
- 保留 1 个真实跳转入口：`日语学习笔记`。
- 展示 `今日待办` 列表（3 条静态项，带状态 badge）。
- 展示 `最近访问` 列表（3 条静态项，带“打开”占位 badge）。
- 展示页脚说明文案。

==== 禁止实现功能（边界）

- 不实现搜索行为（不筛选、不联动、不发请求）。
- 不实现待办的新增、编辑、删除、状态流转与保存。
- 不实现最近访问的点击跳转与排序写入。
- 不接入后端接口，不做鉴权与个性化数据。

==== 输入字段（静态）

[cols="1,3",options="header"]
|===
| 字段 | 说明
| `search.placeholder` | `搜索模块、标签或最近笔记...`
| `modules[]` | 模块卡片列表（图标、标题、描述、最近更新时间、跳转路径）
| `todos[]` | 今日待办列表（标题、说明、状态）
| `recents[]` | 最近访问列表（标题、说明、按钮文案）
| `footerText` | 页脚文案（静态参考页说明）
|===

==== 输出动作

- 点击 `日语学习笔记` 卡片：跳转 `./module/language_jp/index.html`。
- 点击其他 3 个模块卡片：占位动作（`javascript:;`）。
- 其他区块仅展示，不产生数据提交动作。

==== 验收条件

- 首页可完整展示搜索区、4 个模块卡片、待办区、最近访问区、页脚。
- 首页仅 1 个真实跳转入口，其余模块入口均为占位。
- 所有动态能力保持关闭（无接口请求、无状态写入）。

=== P-002 日语学习工作台

原型来源：`static-product/module/language_jp/index.html`

==== 页面定位

日语学习模块工作台，承担“今日学习总览 + 打卡进度 + 分层学习计划 + 模块导航”。

==== 必须实现功能

- 顶部总览卡片：包含返回按钮、标题、完成度、4 个 KPI、新增进度条。
- 今日打卡区：展示 3 项任务（新词、复习、做题）及完成数。
- 学习计划区：展示 N5/N4/N3 三层计划，带标签、目标值、描述。
- 笔记模块区：展示 4 个模块入口（词汇笔记、错词复盘、刷题记录、语法表达）。
- 底部导航：展示 5 个导航项（首页/考试/资料/计划/我的），`首页`为当前激活态。
- 占位导航项点击时弹窗提示 `该功能暂未实现。`。

==== 禁止实现功能（边界）

- 不实现 KPI、打卡、计划的实时计算与写入。
- 不实现笔记模块入口的真实跳转（4 个入口目前均占位）。
- 不实现 `考试/计划/我的` 导航实际页面。
- 不接入后端，不做用户级个性化内容。
- 底部导航配置与激活态由前端管理，后端不返回导航栏定义。

==== 输入字段（静态）

[cols="1,3",options="header"]
|===
| 字段 | 说明
| `overview` | 标题、完成度、KPI（新词/复习/做题/学习时长）、进度条值
| `checkins[]` | 打卡项（名称、描述、状态色、完成进度）
| `plans[]` | N5/N4/N3 学习计划（级别、标签、目标、描述）
| `noteModules[]` | 4 个笔记模块入口（图标、标题、描述、跳转占位）
| `bottomNav[]` | 底部导航项定义（前端本地管理：名称、图标、是否激活、目标路径/占位；后端不返回该字段）
|===

==== 输出动作

- 点击 `← 返回`：跳转 `../../index.html`。
- 点击底部 `首页`：跳转 `./index.html`（当前页刷新）。

== 3. 页面交互逻辑（用户怎么走）

=== 主流程

1. 用户进入 `P-001` 首页并浏览模块卡片、待办和最近访问。
2. 用户点击 `日语学习笔记` 进入 `P-002`。
3. 用户在 `P-002` 点击底部 `资料` 进入 `P-002-M1`。
4. 用户在 `P-002-M1` 可进入子页面（节点编辑/词卡/句型/练习/文章），完成后返回 `P-002-M1`。
5. 用户可通过 `P-002` 顶部 `← 返回` 回到 `P-001`。
6. 用户可单独访问 `P-003` 查看个人中心静态信息（当前与 P-001/P-002 尚未形成真实跳转闭环）。

=== 开发落地顺序建议（避免阻塞）

1. 先落地 `P-002-M1`（资料页），打通 `P-002` 内第一个二级页面真实跳转。
