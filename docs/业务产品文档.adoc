== 2. 页面功能与边界（逐页定义）

=== P-001 首页（基于 `static-product/index.html`）

==== 页面定位

静态产品首页，只负责模块导航、待办展示、最近访问展示。

==== 必须实现功能

- 搜索框展示（占位：`搜索模块、标签或最近笔记...`）。
- 模块卡片区展示 4 个模块卡片。
- 模块卡片包含：图标、标题、描述、最近更新时间。
- “今日待办”区展示 3 条待办及状态徽标。
- “最近访问”区展示 3 条最近访问记录及打开标记。
- 页面底部展示静态参考说明。

==== 禁止实现功能（边界）

- 不做真实搜索请求与搜索结果联动。
- 不做待办编辑、删除、保存。
- 不做最近访问写入与排序。
- 不接入后端接口。

==== 输入字段（静态）

[cols="1,3",options="header"]
|===
| 字段 | 说明
| `searchPlaceholder` | 搜索框占位文案
| `modules[]` | 模块卡片列表（图标、标题、描述、更新时间、跳转路径）
| `todos[]` | 今日待办列表（标题、描述、状态）
| `recentVisits[]` | 最近访问列表（标题、描述、状态）
| `footerText` | 页脚说明文案
|===

==== 输出动作

- 点击“日语学习笔记”模块卡片：跳转 `./language-jp.html`（当前页面内定义）。
- 点击其他 3 个模块卡片：当前为占位动作（`javascript:;`）。
- 其他区域仅展示，无提交类动作。

==== 验收条件

- 页面加载后可看到搜索区、模块区、今日待办区、最近访问区、页脚。
- 4 个模块卡片展示信息完整。
- 仅存在 1 个真实跳转入口，其余入口保持占位。
- 页面全流程可展示但不依赖后端。

== 3. 页面交互逻辑（用户怎么走）

=== 主流程

1. 用户进入 `P-001` 首页。
2. 用户查看模块区、今日待办、最近访问。
3. 用户点击“日语学习笔记”进入下一页面；或停留在首页查看信息。
