# language-jp 节点新增/编辑改造方案

- 文档日期：2026-04-16
- 适用范围：`note-ui` 前端
- 目标页面：`/language-jp/materials/*`
- 参考原型：`static-product/module/language_jp/node_editor/index.html`

## 1. 背景与目标

当前节点详情页已有“新增/编辑”入口，但功能状态不一致：

1. 详情页头部“编辑”可跳转到编辑页，但保存逻辑未接入。
2. 子节点区域“新增”“编辑”仅有占位提示，尚未进入真正流程。

本次目标：

1. 复用现有 `edit.vue`，改为“新增/编辑同页”。
2. 页面样式和交互对齐参考原型（布局、固定底部操作区、父节点搜索、标签选择）。
3. 打通新增与编辑保存链路（前端路由 + API 调用 + 跳转闭环）。

## 2. 实施方案（可执行）

### 2.1 路由与模式设计

在现有路由上增加“新增节点”路由，形成两种模式：

1. 新增模式：`/language-jp/materials/new?parentId={id}`
2. 编辑模式：`/language-jp/materials/:id/edit`

页面内通过路由参数判断模式，统一使用 `NoteEditView`。

### 2.2 详情页入口改造

文件：`note-ui/src/views/commons/notes/index.vue`

1. `handleAddChild()`：跳转新增路由，默认 `parentId=当前节点id`。
2. `handleEditChild(item)`：跳转编辑路由，`id=item.id`。
3. `openEditPage()`：继续用于编辑当前节点。

### 2.3 编辑页结构与样式改造

文件：`note-ui/src/views/commons/notes/edit.vue`

按参考原型对齐以下 UI/交互：

1. 顶部卡片：标题、返回按钮、副说明文字。
2. 表单区：
   1. 节点名称
   2. 父节点搜索选择（combobox + 下拉）
   3. 标签编辑（chip + 删除 + 搜索候选）
   4. 类型
   5. 顺序
3. 固定底部操作区：取消/保存。
4. 移除纯占位文案“保存（暂未接入）”，改为真实提交状态（保存中/保存成功/保存失败）。

说明：内容 `contentText` 字段可保留在高级区，或根据业务决定是否在首版显示。

### 2.4 数据流与提交流程

#### 编辑模式

1. 进入页面后加载节点详情，回填表单。
2. 修改后点击保存，调用更新 API。
3. 成功后跳转回节点详情页并触发刷新。

#### 新增模式

1. 带 `parentId` 进入页面，初始化空表单。
2. 保存时调用创建 API。
3. 成功后跳转到“父节点详情页”。

### 2.5 交互与状态规范

1. 表单校验失败：页内提示，不使用 `alert`。
2. 请求失败：在状态区展示后端错误信息。
3. 保存按钮防重复点击：提交中禁用。
4. 取消按钮：返回来源页；若无来源则回到父节点详情。

### 2.6 验收标准

1. 新增和编辑均能进入同一页面并正确区分模式。
2. 页面视觉和布局风格与参考原型一致。
3. 父节点搜索和标签选择可用（至少有基础可选能力）。
4. 保存操作真实调用 API，成功后页面可见更新结果。
5. 无新增 lint/build 错误。

   2. 改造分提交，必要时仅回退 `edit.vue` 逻辑层。
